可以实现，而且关键不是“把物品随机化”，而是把**线索的承载方式随机化**。

你现在会固定下来，根因有三层：

第一，`scene-gen.ts` 里场景先被限定为固定屋型，而且物品大部分还必须来自屋主日常生活与职业。第二，物品总数只有 8–12 个，还要求至少 3 个 `true_clue`、至少 2 个 `beadReactive`。第三，prompt 明确要求“先确认 5 条推理线都有物品承载，再填充氛围物品”。在这种强约束下，模型最容易收敛到一套“最稳、最省力”的固定配方，也就是你观察到的“固定物品承担固定线索”。

你这件事确实不适合做成“简单随机选物品”。因为你文件里已经把 5 条推理线和承载类型绑得很清楚了，尤其是 B 线只能落在 `true_clue`，E 线只能通过 `beadReactive` 的残念呈现，所以乱随机很容易把逻辑打坏。

---

## 最合适的思路

把“线索”从“具体物品”里拆出来，先变成一层**证据功能**，再让模型去决定“哪种东西来承载这个功能”。

也就是从：

* A 线 = 药碗
* B 线 = 倒桌子
* C 线 = 门口脚印
* D 线 = 骨粉
* E 线 = 某个残念物

改成：

* A 线 = “能证明死者死前已异常”的证据功能
* B 线 = “能证明屋内有过激烈冲突”的证据功能
* C 线 = “能证明有外来者来过”的证据功能
* D 线 = “能证明有东西被带走”的证据功能
* E 线 = “能证明外来者并非纯粹攻击者”的证据功能

然后每条功能再从一个“承载家族”里随机选实现方式。

这就像你 `real_random_trick.md` 说的，不是让 LLM 凭空随机，而是给它**不同的上文锚点**，让它沿不同路径展开。这里只不过随机的不是“最终物品名”，而是“证据应该以什么形式出现”。

---

## 你应该新增一层：线索规划层

在真正生成 `items` 之前，先让模型在内部做一份不输出给玩家的“线索承载规划”。

这层不要直接写物品名，而写 5 个字段：

* `line`：A/B/C/D/E
* `carrier_family`：承载家族
* `manifestation_mode`：显现方式
* `host_domain`：它依附在哪类生活对象上
* `category_rule`：必须是 `true_clue` 还是 `beadReactive`

### 这层的意义

同一条线每次都能换“承载形式”，但逻辑仍然稳定。

比如 A 线不再总是药碗，它可以是：

* 治疗痕迹
* 发病体征痕迹
* 照护/约束痕迹
* 被中断的治疗动作

这样你得到的是“更丰富的线索”，不是“更随机的杂物”。

---

## 五条线各自的“承载家族”

下面这组最适合你的项目。

### A 线：死者死前状态异常，疑似先发病

不要只写“药碗/药方”。改成 4 个承载家族：

* **治疗物**：药碗、药包、残方、煎药器具
* **身体体征物**：污布、抓痕褥席、带齿痕的木片、呕吐残留容器
* **照护/约束物**：束带、压皱被褥、固定用的绳结、安神香灰
* **被中断的诊治动作**：翻开的医书、半磨的药末、未喝完的药汁

这样 A 线每次都还在，但未必总是“药碗”。而且这仍然符合你原文里 A 线允许由 `true_clue` 或 `beadReactive` 承载。

### B 线：屋内曾发生激烈冲突

这一条本来就更物理，所以仍然应该偏具体痕迹，但不要总是“倒桌 + 血迹”。

可以拆成：

* **家具受力痕迹**：撞痕、裂角、翻倒、卡进墙缝
* **挣扎位移痕迹**：拖拽褥席、歪斜板凳、散落小件
* **防御/反击痕迹**：被抓起又丢开的工具、护身时留下的破损
* **打击结果痕迹**：木屑、碎陶、擦血、钝器印

注意 B 线仍然必须落在 `true_clue`，这个不要变。

### C 线：有外来者来过现场

不要总是门口脚印。可以拆成：

* **入口异常**：门槛、窗台、门闩、窗纸、门边泥屑
* **外来材质**：不属于屋主生活圈的纤维、灰粉、皮屑、草籽
* **借道痕迹**：本不该移动的位置被踩踏、搭脚、攀附
* **停留痕迹**：临时搁放、扶按、轻触后留下的异样

这一条你原文允许 `true_clue` 或 `beadReactive`，所以可以让有的局走“物证”，有的局走“残念感知”。

### D 线：现场有骨骼/物品被带走的迹象

不要总是骨粉 + 拖痕。拆成：

* **搬运路径**：拖痕、负重擦痕、门窗方向的连续位移
* **缺失痕迹**：被掏空的箱格、空出的墙钩、缺了一样该在场的东西
* **残留碎屑**：骨粉、碎片、捆绑残线、包裹残屑
* **二次整理痕迹**：为了拿走某物而翻动、挪开、再勉强摆回

这一条也允许 `true_clue` 或 `beadReactive`，所以既能落在“明显物证”，也能落在“念珠感到某处本该有东西”。

### E 线：外来者行为不像“食人妖怪”

这一条最容易单调，因为它只能通过 `beadReactive` 呈现。

所以你不要让它固定成某个“残念物”，而要给它几个不同的**温和动作模板**：

* **照护动作**：替患者垫高、扶正、掩盖、盖衣
* **迟疑动作**：手伸到一半又缩回、停顿、回头、避开致命处
* **悲悯动作**：收拾散落遗物、把伤者摆正、轻放、停留
* **克制动作**：明明能粗暴处理却选择绕开、小心搬动、避免惊扰

然后再让这些动作附着到不同宿主上：

* 被轻轻掖好的被角
* 被扶正的小凳
* 被移到一旁免得绊脚的盆
* 被遮上的脸布
* 被重新合起的门板

这样 E 线就会丰富很多，而且不会失真。

## 真正让它“随机起来”的不是物品名，而是这三层

你可以把每条线的变化拆成三层随机：

### 1. 承载家族随机

先决定这条线是“治疗物”还是“身体体征物”，是“入口异常”还是“外来材质”。

### 2. 显现方式随机

同样是 A 线，也可以是：

* 直接证据：一眼看得出
* 间接证据：需要联想
* 残留证据：事件之后留下的边角
* 负空间证据：某个本该存在的东西不见了

### 3. 宿主物件随机

同样是“照护痕迹”，可以落在：

* 褥席
* 被褥
* 矮凳
* 药罐
* 布巾
* 门边

这样最终呈现出的“物品”会不同，但线索逻辑仍然成立。

---

## 最关键的一条规则

**不要要求“一条线 = 一个固定物品”。**

改成：

* 一条线由“一个主承载物”负责
* 允许再有一个“辅承载物”增强说服力
* 单个物品最多承载 2 条线
* B 和 E 尽量不要和别的线共用宿主
* A/C/D 可以适度交叉

这是很有用的，因为现实里的证据本来就会交叠。

例如：

* 一个“门边泥印”可以既服务 C 线，也暗示 D 线的搬运方向
* 一个“被掖好的被角”可以服务 E 线，同时弱弱呼应 A 线的照护情境
* 一个“翻倒又蹭裂的矮柜”可以服务 B 线，并在柜中缺失物件时顺带呼应 D 线

这样会比“五条线五个固定物”自然很多。

## 最适合你的实现方式

### 方案一：单节点，但增加“隐藏规划”

仍然用现在这个节点，只是在 prompt 里加一段：

```text
在生成最终 items 之前，请先在内部完成“线索承载规划”，但不要输出规划过程。

对 A/B/C/D/E 五条线分别确定：
1. 承载家族（carrier_family）
2. 显现方式（manifestation_mode）
3. 宿主领域（host_domain，如床铺、门口、灶边、箱柜、工具、衣物、器皿）
4. 最终落在 true_clue 还是 beadReactive
5. 是否与其他线共享同一宿主（最多允许两条线共享）

要求：
- 不要默认使用高频承载物（如药碗、门口脚印、骨粉、倒桌）
- 优先从“痕迹类型”出发，再决定由哪件东西承载
- 至少3条线必须使用“非默认承载方式”
- 每次生成时，应让 A/C/D/E 的承载家族尽量与上一次不同
```

这个改动最小。

### 方案二：双节点，效果最好

先单独生成一个 `clue_plan`，再让第二个节点根据 `clue_plan + house_seed` 生成 items。

这是最稳的，因为你现在的主节点已经承担了：

* 屋型一致
* 场景文本
* zones
* items
* iconTag
* 5 条推理线覆盖

任务太多时，模型一定会走最省力模板。

把“线索规划”单独拆出来，变化会明显大很多。

## 随机串应该怎么接进这件事

不要再把随机串只用于“房屋类型”。
应该额外给“线索承载规划”一个随机锚点。

例如：

* `sceneSeed`：决定住所类型与生活面貌
* `clueSeed`：决定 5 条线各自的承载方式

比如 `clueSeed = M7QXK`

你不让模型直接解释成物品，而是解释成：

* 第1位：A线承载家族偏好
* 第2位：B线显现强度
* 第3位：C线更偏入口/材质/路径
* 第4位：D线更偏缺失/搬运/碎屑
* 第5位：E线更偏照护/迟疑/悲悯/克制

这就把 `real_random_trick` 真正用到了“线索结构”上，而不是只用在表面文案上。

## 一段更适合你当前 prompt 的替换文案

你可以把现有那段“极其重要”之后，再加下面这段：

```text
为了避免线索物品固化，请不要将每条推理线机械地绑定到固定高频物品。
你必须先把每条推理线抽象为“证据功能”，再决定由什么物件、痕迹或残念来承载。

在内部规划时，每条推理线都要先确定：
- 承载家族：治疗物 / 身体痕迹 / 家具受损 / 入口异常 / 外来材质 / 缺失痕迹 / 搬运路径 / 照护动作 / 迟疑动作 / 悲悯动作 等
- 显现方式：直接物证 / 间接痕迹 / 残留状态 / 缺失状态 / 念珠残念
- 宿主领域：床铺 / 门口 / 窗边 / 箱柜 / 炉灶 / 工具 / 衣物 / 器皿 / 墙面 / 地面

约束：
- B线必须是可见的物理冲突证据，并落在 true_clue
- E线必须通过 beadReactive 呈现，且体现小心、犹豫或悲悯
- A/C/D线不要默认使用药碗、脚印、骨粉这类最高频答案，除非本次屋型与事件逻辑强烈支持

```

这段会明显改善“固定线索 = 固定物件”的问题。

## 我的结论

你这个问题的正确解法不是“让物品更随机”，而是：

**让每条推理线先随机到不同的证据形态，再由物品去承载这些证据。**

一句话概括就是：

**把随机性放在线索的“表现形式”上，而不是放在物品名上。**

这样既不会破坏你现在这套很重要的 5 条推理线约束，仍然符合 `true_clue / beadReactive / house type / items 数量` 这些硬规则，变化又会明显增多。

下一步最有效的做法，是我直接帮你把 `scene-gen.ts` 里“物品创意方向”这一段改写成一版可直接替换的 prompt。
