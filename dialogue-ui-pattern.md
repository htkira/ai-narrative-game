# 文字演出部分的显示和交互

请把这个组件理解为“固定双区对话框”，而不是聊天列表。这是一种“视觉小说 / 互动小说”式对话界面，其中上半区是 NPC / 旁白文本展示区，使用打字机效果逐字出现；下半区是玩家可点击的台词/选项区，文字一次性完整显示，不使用打字机效果。

### 1. 布局结构

* **上半部分**：显示旁白或 NPC 的台词。
* **中间**：有一条横向分割线，把“上方叙事区”和“下方交互区”分开。
* **下半部分**：显示主角的话 / 玩家可点击的选项。

### 2. 文字显示规则

* **上半部分的文字**必须使用 **打字机效果**：

  * 文字是**逐个字出现**的，不是整段一次性出现。
  * 效果类似 ChatGPT 回复时那种流式输出，但这里按“字符逐个显示”实现。
  * 只对 **NPC / 旁白文本** 生效。
* **下半部分的文字**不要打字机效果：

  * 玩家可点击的台词 / 选项要**一次性完整显示**。
  * 不能一个字一个字冒出来。

### 3. 交互逻辑

* 当前段落中：

  * 上方显示 NPC / 旁白内容。
  * 下方显示玩家当前可点击的一句台词，你可以把它理解为选项，但是只有这一个选项。
* 当玩家**点击下方主角台词**后：

  * 进入下一段对话。
  * 上方内容切换为下一段 NPC / 旁白文本，并重新执行打字机效果。
  * 下方内容切换为下一段玩家台词 / 选项，并直接完整显示。
* 也就是说，这个界面本质上是：
  **“上方播报剧情，下方点击推进下一句”** 的流程。

### 4. 视觉反馈

* 下方玩家台词 / 选项在鼠标悬停或选中时，需要有明显的高亮/变色反馈。
* 默认状态下它看起来像一条可点击的文本选项，而不是普通静态文本。
* 点击后立即切换到下一段内容。

### 5. 需要避免的误解

请不要把它做成以下几种错误形式：

* 不是聊天软件那种上下连续堆叠消息流。
* 不是上下两边都打字机显示。
* 不是点击整个对话框推进，而是**点击下方玩家台词/选项推进**。
* 不是让玩家手动输入文本，而是点击预设好的主角话语/选项。
* 不是“上方是描述，下方是按钮 UI”，而是“下方本身就是主角说的话/可选回应文本”。

### 6. 更准确的定义

这是一种：

* **视觉小说式对话框**
* **上方 NPC/旁白逐字播放**
* **下方玩家文本选项立即显示**
* **点击下方文本推进到下一段剧情**
  的交互模式。
